あなたは要件定義のプロです。私はwebエンジニアです。

自分専用のinboxアプリを作りたいと思っています。

やらないといけないことはたくさん降ってくるし、やりたいことも湧き出てくる。なにかのタスクに集中してるとやるべきことを忘れてしまうという課題を解決したいです。

本日のtodoがinboxとして入ってくる機能と、いつかやりたいことを積んでおく機能を2カラムで表示し、「今日やるべきこと」と「今日できたらやりたいこと」を一目で管理できるツールを作りたいです。
Todoは期限付きのタスク。Inboxは期限がないけどとりあえず積んでおきたいタスクを書きためる。

機能一覧
- ログイン機能(google認証)
- トップ画面（本日やることとやりたいことが一覧で表示される）
- 設定機能（ログアウト）

Inbox機能の要件
- いわゆる本日のtodoリスト
- 未来が期日のタスクも追加できるが、表示されるのは本日の分のみ。
- 新しく追加したものは一番下に追加されるキュー構造。
- 意図としては今日が期限のタスクに集中することを目的としています。
- キュー内は並び替えができます。
- Todoには未着手、進行中、相手待ち、完了のステータスがあります。
- 期限が過ぎたものは翌日に持ち越されます。todoリストからは消えません。
- todoを作成するとデフォルトでは期日が本日になる。もちろん自分で期日を設定することもできる。
- デフォルトでは本日のタスクが表示されるが、Inbox側のカラムは日付をページネーションで移動することができる
- アイテムを削除することができる

やりたいことリスト機能の要件
- 新しく書いたものが上に表示されるスタック構造。
- Inboxのアイテムに期限を設定したらTodo欄に移動する。
- Todoアイテムの期限を消したらInboxに戻る。期限はなしになる。
- 期日はないため、毎日同じものが表示される
- キュー内は並び替えができる
- アイテムを削除することができる
